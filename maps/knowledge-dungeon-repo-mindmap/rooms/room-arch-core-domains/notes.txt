# Architecture & Core Domains

## The Five Core Modules

Knowledge Dungeon's domain logic lives in `/src/core/`. These modules are pure TypeScript with no dependencies on React, Electron, or game logic. They can be ported to any platform.

### Overview of Each Domain

1. **Graph Module** — Manages the dungeon structure (rooms, topics, edges)
2. **Validation Engine** — Checks if notes meet quality standards
3. **Progression System** — Calculates XP, ranks, and badges
4. **Artifacts & Review** — Generates markdown artifacts and tracks review passes
5. **Persistence Types** — Shared domain types and schemas

## Design Philosophy

Each module is **pure domain logic** that:

- Has no dependencies on React, Phaser, or Electron
- Uses functional programming with explicit error handling (Result<T, E> pattern)
- Maintains clear separation of concerns
- Can be tested with Vitest without browser/DOM

## Why This Matters

By keeping domain logic separate, we can:

- **Migrate to a new UI framework** without rewriting the core
- **Port to new platforms** (web, mobile, desktop) with only a new shell
- **Unit test everything** without mocking browser/game APIs
- **Understand the app** by reading pure TypeScript, not mixed with UI code
