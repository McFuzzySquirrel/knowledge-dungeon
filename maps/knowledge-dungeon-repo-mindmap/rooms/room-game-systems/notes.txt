# Game Systems

## Overview

Game systems are Phaser-based systems that render the dungeon and handle player interaction. They bridge the domain logic with the game world.

## The Three Core Systems

1. **Dungeon Rendering** — Draw the map, rooms, walls, player
2. **Encounter System** — Detect collisions with rooms, trigger modals
3. **Controls & Navigation** — Keyboard/touchscreen input
4. **Room Panel & HUD** — React-based overlays showing state

## Phaser: The Game Engine

Phaser 3 is a 2D JavaScript game framework:

- **Canvas rendering** — Fast 2D graphics on GPU
- **Sprite management** — Draw and update game objects
- **Physics system** — Collision detection, movement
- **Event system** — Fire events when things happen
- **Scene system** — Organize game logic into scenes

### Scene Architecture

A "scene" is a self-contained game state (the dungeon, menus, etc.):

- `DungeonScene` — Main game world, player movement
- `DungeonHUD` — Overlay with phase/rank/XP
- `RoomPanel` — Shows current room description

Each scene has:
- `preload()` — Load assets
- `create()` — Set up game objects
- `update()` — Called every frame, updates position/state
- Event listeners for input

## Data Flow

```
Zustand Store (current room, phase)
  ↓
DungeonScene.update()
  ↓
Render rooms, player position
  ↓
Player presses E on a room
  ↓
Emit event to open encounter
  ↓
React modal appears (form for note submission)
  ↓
On submit, validation runs (domain logic)
  ↓
Update Zustand store with result
  ↓
Phaser re-renders updated game state
```

## Visual Design

Rooms are rendered as **connected nodes** in a graph-like layout:

- Each room is a visual tile
- Edges are drawn as connecting lines
- Player is a simple character sprite
- Colors indicate room state (uncreated, created, cleared, etc.)

## Design Insight

Game systems are **pure rendering and I/O**. They know how to draw things and handle input, but don't know about learning, validation, or progression. This separation keeps concerns clear: domain logic in core/, presentation in game/.
