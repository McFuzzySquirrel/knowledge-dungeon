# Development Workflow

## Overview

This section documents how Knowledge Dungeon is **built, tested, and distributed**. These practices keep the codebase healthy as it grows.

## The Development Cycle

```
1. Make changes in src/
2. Hot reload in dev browser
3. Run tests locally (npm run test)
4. Run linting locally (npm run lint)
5. Commit with git
6. Push to GitHub
7. GitHub Actions runs full CI
8. Merge to main
9. Build release artifacts
```

## Key Commands

### Development
```bash
npm install          # Install dependencies
npm run dev          # Start Vite dev server
npm run electron     # Build web + launch Electron
```

### Quality Checks
```bash
npm run lint         # Run ESLint
npm run typecheck    # TypeScript type check
npm run test         # Run Vitest
npm run test:watch   # Run tests in watch mode
```

### Building
```bash
npm run build:web    # Build web bundle
npm run build:electron  # Build web + Electron main
npm run package:electron    # Package desktop app (current OS)
npm run package:electron:mac # macOS DMG + ZIP
npm run package:electron:win # Windows NSIS installer
npm run package:electron:linux  # Linux AppImage + deb
```

### Utilities
```bash
npm run check:bundle-size  # Verify bundle doesn't exceed limit
```

## Development Workflow Best Practices

### 1. Start dev server
```bash
npm run dev
```
Vite launches on http://localhost:5173. Changes hot-reload instantly.

### 2. Make changes
- Edit source files
- Browser refreshes automatically
- TypeScript errors show in console
- ESLint warnings shown in terminal

### 3. Test frequently
```bash
npm run test -- --watch
```
Keep test runner open in another terminal. Tests re-run on file changes.

### 4. Lint before commit
```bash
npm run lint --fix
npm run typecheck
```
Auto-fix formatting and check types. Commit when all pass.

### 5. Commit with meaningful messages
```bash
git commit -m "feat: add cross-links to graph module"
```
Use conventional commits (feat:, fix:, docs:, test:, refactor:).

## CI/CD Pipeline

### GitHub Actions (`.github/workflows/`)
On every push:

1. **Lint** — ESLint checks code style
2. **Type Check** — TypeScript compiler checks types
3. **Test** — Vitest runs all tests
4. **Bundle Check** — Verify bundle size stays under limit

If all pass, PR is ready to merge. If any fail, you get detailed error feedback.

## Design Insight

The dev workflow is **designed for fast feedback**. Hot reload means you see changes in <1 second. Tests run in parallel, type checking is instant. This rapid feedback loop makes development enjoyable and catches bugs early.
