# Frontend: React & Phaser

## The Two-Layer Approach

Knowledge Dungeon uses React and Phaser together, each for what it does best:

### React Layer
Handles:
- Home screen (subject selection, creation)
- Modals (room creation, note submission)
- HUD (phase indicator, rank, XP counter)
- Room panel (topic description, notes form)
- Help and settings screens

### Phaser Layer
Handles:
- Dungeon rendering (tiles, walls)
- Player character and camera
- Room visuals and positioning
- Encounter animations
- Minimap rendering

## Why This Split?

**React** is great for forms, dialogs, and reactive UI. **Phaser** is great for 2D graphics and keyboard input handling. Together, they avoid forcing one paradigm onto the other.

React renders a DOM; Phaser renders a Canvas. They coexist by:
- Phaser canvas fills the center of the screen
- React panels and HUD surround it
- Event listeners bridge them (keyboard, clicks on HUD buttons trigger game state)

## Integration Points

### Player Movement
- Phaser detects WASD/arrow keys
- Updates player position in the game world
- Zustand stores track current room

### Encounter Trigger
- Player collides with room (Phaser physics)
- Modal opens (React) showing encounter
- Note submission triggers validation (domain logic)
- Result updates game state (Zustand) and re-renders

## Bundle Size Consideration

React + Phaser is ~150KB gzipped. For a study app, this is acceptable trade-off for UX clarity over minimal bundles.
