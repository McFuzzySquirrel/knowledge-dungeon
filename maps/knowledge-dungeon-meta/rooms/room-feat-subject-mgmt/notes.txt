# Subject Management

## Creating a New Subject

1. **On home screen**, click "Create Subject"
2. **Enter root topic** (e.g., "React", "Machine Learning")
3. **Confirm creation** — Subject is initialized in Creator phase
4. **Enter dungeon** — Start building the topic map

### Under the Hood

`createRootDungeon()` in graph module:

```typescript
const dungeon = {
  dungeonId: generateId(),
  subjectName: "React",
  rootRoomId: "room-1",
  rooms: [{ roomId: "room-1", topic: "React", state: "Created" }],
  edges: [],
  phaseState: "SubjectCreated"
};
```

A new subject starts with just the root room, no edges.

## Loading a Subject

On home screen, you see:

- **Subject list** — Name, room count, creation date
- **Refresh button** — Re-scan disk for new subjects (Electron)
- **Load button** — Open selected subject in the dungeon

### Storage Locations

**Web (localStorage)**
- Key: `dungeon-{subjectId}`
- Value: Full SubjectSnapshot JSON
- Limited to ~5MB per domain

**Electron (filesystem)**
- Path: `<userData>/dungeon-data/{subjectId}/dungeon.json`
- Includes: dungeon.json + rooms/ directory with notes and artifacts
- Backups in `.backups/` folder with timestamps

## Exporting/Importing

### Export
- **Web**: Download dungeon.json + rooms as ZIP
- **Electron**: Admin menu → "Export Subject" → choose folder

### Import
- **Web**: Upload ZIP file with dungeon.json
- **Electron**: Admin menu → "Import Subject" → choose folder

This enables **sharing subjects** between machines or collaborators.

## Deleting a Subject

- **Confirm first** — Prevents accidental loss
- **Permanent** — No undo (though backups exist if available)
- Only available on subjects you own/created locally

## Design Insight

Subject management is **simple and forgiving**. You can create, load, and export subjects without leaving the app. Backups and import/export ensure you never lose work. The home screen is a dashboard where your learning is visible and organized.
