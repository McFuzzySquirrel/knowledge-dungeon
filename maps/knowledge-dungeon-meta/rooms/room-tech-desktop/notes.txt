# Desktop: Electron

## What Electron Does

Electron packages a web app as a native desktop application:

- **Main Process** — Runs in Node.js, handles file system, native menus, window management
- **Renderer Process** — Runs the web app (React + Phaser) in a Chromium window
- **Preload Bridge** — Secure channel between web code and native code

## Why Desktop?

Knowledge Dungeon works on the web, but **desktop offers advantages**:

1. **File system access** — Save/load subjects to disk directly
2. **Native menu** — macOS, Windows, Linux consistent menus
3. **Offline-first** — No internet needed (works fully offline)
4. **Desktop features** — Dock integration, file associations, auto-update
5. **App feel** — Behaves like a "real" app, not a web page

## Key Electron Components

### Main Process (`electron/main.ts`)
- Creates windows
- Handles file operations (reading/writing subject folders)
- Responds to menu commands

### Preload Script (`electron/preload.ts`)
- Exposes safe APIs to React code
- Example: `window.electronAPI.loadSubjects()` reads the dungeon-data folder

### Build Process
- Vite builds React app as usual
- Electron bundles the built assets
- electron-builder creates platform installers

## Packaging Targets

| Platform | Format | Command |
| --- | --- | --- |
| macOS | `.dmg` + `.zip` | `npm run package:electron:mac` |
| Windows | NSIS installer + `.zip` | `npm run package:electron:win` |
| Linux | `.AppImage` + `.deb` | `npm run package:electron:linux` |
| All three | All of above | `npm run package:electron:full` |

## Design Insight

Electron makes Knowledge Dungeon feel like a professional app while keeping the codebase a single web app. The main/preload separation ensures the web code stays sandboxed and secure.
