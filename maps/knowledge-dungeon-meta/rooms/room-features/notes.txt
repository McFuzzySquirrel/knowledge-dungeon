# Features & Mechanics

## Feature Overview

This room summarizes the main user-facing features. See linked subtopics for details.

## Core Features

### Subject Management
- Create new subjects (start with a root topic)
- Load existing subjects from localStorage or disk
- Delete subjects (with confirmation)
- Import/export subjects (web & desktop)
- Search subjects by name

### Note Validation
- Automatic quality checking on submission
- Detailed feedback on what passed/failed
- Manual override option ("I'm confident" button)
- Unlimited resubmission until you pass

### Quality Scoring
- Five rubric criteria for balanced assessment
- Transparent scoring (you see every score)
- Quality bonus that feeds progression
- Motivates better learning through rewards

### Artifact Generation
- Markdown document with your notes + scores
- Generated automatically on room clear
- Stored for review later
- Build a portfolio of learning

### Review Mechanics
- Revist artifacts during Archaeologist phase
- Answer self-check prompts
- Track review streaks
- No penalty for reviewing multiple times

### Progression Tracking
- XP earned per cleared room
- Rank advancement (Novice → Sage)
- Badges for phase milestones
- HUD shows current progress

## Why These Features?

These features support the **learning loop**:

1. **Create** ← Subject management
2. **Explore** ← Encounter system
3. **Validate** ← Note validation + quality scoring
4. **Record** ← Artifact generation
5. **Review** ← Review mechanics
6. **Progress** ← Progression tracking

Together, they create a complete study experience.

## Design Philosophy

Every feature serves learning. Flashiness is avoided—the focus is **content and clarity**. Progress is visible, feedback is immediate, and motivation comes from intrinsic mastery goals, not external points.
